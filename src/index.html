A (nearly) pointless demo

I picked an entirely arbitrary concept to demonstrate my CSS, React, and JavaScript skillset. Why pokemon—and why are they spinning?

Well, a couple of reasons

  • There's a free Pokemon api, which is cool. It was useful to demonstrate REST API integration.
  • I had been thinking about how to recreate some retro/pixel interface elements in CSS. Specifically utilizing step transition, dither effects, and simulating pixelation.
    • Note the segmented fill and dither effects of the stat bars. I spent entirely too much time calculating the transition delays by comparing the ranges of the changed indicies, but I'm happy with the result.
    • The aliased edges of the buttons is done using CSS's clip-path property.
  • But why are they spinning? I had been noodling on this idea for a while and wanted to see if it would come together. Once I had the inital proof of concept down I wanted to push it a little further, to see if I could smoothly add items into it without disrupting the animation. Click "Add Item" to make an asyncronous request for a random Pokemon. One nuance that might not be apparent at first is how each item has to counter-rotate in sync with the disc in order to always be facing towards the user. All of the rotation animations are done using a function of the item's index and the number of items in the carousel.

All in all, its a pretty dumb little component that demonstrates a range of front-end skills.
